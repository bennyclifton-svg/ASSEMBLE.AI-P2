## Agent Brief

**Category:** enhancement
**Summary:** Wire Polar checkout, portal, and idempotent subscription webhook sync to the public SaaS source of truth.

**Current behavior:**
Public pricing and billing UI can call Better Auth Polar checkout by slug, and Better Auth can expose Polar checkout/portal/webhook endpoints when configured. Local subscription state is not yet fully synchronized from validated Polar webhooks, and billing UI/lookup only partially reflects Better Auth user plus Polar customer/subscription records.

**Desired behavior:**
Starter and Professional checkout should be created from the shared public plan catalog. Customer portal access should be available whenever the authenticated Better Auth user has a Polar customer. Validated Polar webhook payloads should upsert local Polar subscription rows idempotently so duplicate events do not create duplicate or contradictory subscription state. Active, canceled, and past-due subscription states should flow into the entitlement and billing read paths.

**Key interfaces:**
- Checkout request builder - validates requested public plan and returns the Better Auth Polar checkout payload.
- Portal eligibility helper - answers whether a user can open the customer portal from local Polar customer state.
- Polar webhook sync service - accepts a signature-validated webhook payload and idempotently upserts subscription state.
- Billing subscription API - maps the current Better Auth user to Polar customer/subscription records and returns plan/status data for the billing UI.

**Acceptance criteria:**
- [ ] Starter and Professional checkout requests are generated from the shared plan catalog.
- [ ] Billing subscription lookup maps the authenticated Better Auth user to their Polar customer and subscription state.
- [ ] Customer portal UI is available for users with a Polar customer, including trial users.
- [ ] The webhook sync path only changes local state after signature validation has happened.
- [ ] Duplicate subscription webhook events remain idempotent.
- [ ] Subscription created, updated, canceled, and past-due events update local subscription state.
- [ ] Entitlement state can consume active, canceled, and past-due states from synced Polar rows.

**Out of scope:**
- Adding metered usage to Polar.
- Replacing the Better Auth Polar plugin.
- Sending billing emails.
- Enforcing entitlements across all mutating API routes.

## Implementation notes

- Checkout requests now validate Starter/Professional intent against the shared public catalog before calling Better Auth Polar checkout.
- Billing subscription lookup returns entitlement-backed plan/status plus whether the current Better Auth user has a Polar customer.
- Customer portal UI appears for any authenticated user with a Polar customer, including trial users.
- Better Auth Polar's validated webhook callback now calls an idempotent local subscription sync service.
- Subscription created, updated, canceled, and past-due payloads upsert the Better Auth Polar subscription table.
