## What to build

Add the first concrete trial cap by enforcing one project for trial users and requiring authenticated workspace ownership for project creation and project selection. This gives the SaaS access-control model a narrow end-to-end proof point before the broader API hardening pass.

## Agent Brief

> *This was generated by AI during triage.*

Build the project cap as a small, reusable gate on top of the entitlement service and plan catalog. Use authenticated user/workspace state from Better Auth only; do not accept ownership from the client. Apply the gate to the existing project creation endpoints, preserve workspace-scoped project reads, and add focused tests for trial cap behavior, paid-plan allowances, and workspace isolation.

## Acceptance criteria

- [x] Trial users can create the first project in their own workspace.
- [x] Trial users are blocked from creating a second project unless their plan allows it.
- [x] Project creation derives the workspace/user from the Better Auth session rather than trusting client-supplied ownership.
- [x] Project list and selection only return projects owned by the current user's workspace.
- [x] Blocked project creation uses the shared entitlement/upgrade response.
- [x] Tests cover first project creation, second project block, paid-plan allowance, and cross-user project isolation.
