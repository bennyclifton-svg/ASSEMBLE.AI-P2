## Agent Brief

**Category:** enhancement
**Summary:** Add a central SaaS entitlement state and visible trial/expired-trial banner.

**Current behavior:**
Signup now records trial metadata on the Better Auth user record, and Polar subscription tables exist, but access decisions are still split across older subscription helpers and individual UI/API areas. The app shell does not consistently show active trial or expired trial state.

**Desired behavior:**
The app should have a centralized entitlement interface that derives a user's access state from Better Auth trial metadata and Polar subscription data. The result should answer whether the user can read data, write/edit, upload, run AI actions, export/download, and manage billing. Active trial and active paid subscription users should be writable. Expired trials, canceled subscriptions, past-due subscriptions, and users missing an active subscription source should remain able to read owned data and export/download, but should not be considered writable. The app shell should surface active-trial and expired-trial state with clear billing CTAs.

**Key interfaces:**
- Entitlement evaluator - accepts trial fields and optional subscription data and returns a stable access-state object.
- Current-user entitlement loader - loads the authenticated user's Better Auth row and Polar subscription row, then delegates to the evaluator.
- App shell banner - renders active-trial and expired/read-only states using the entitlement result.
- Blocked-action messaging - reusable copy for later issues to show when a write/upload/AI/export/billing action is denied.

**Acceptance criteria:**
- [ ] The entitlement result exposes read, write, upload, AI action, export, and billing-management allowances.
- [ ] Active trial users are writable within the entitlement result.
- [ ] Expired trial users can read/export/manage billing but cannot write/upload/run AI.
- [ ] Active paid subscription users are writable.
- [ ] Canceled, past-due, and missing subscription states are represented and covered by tests.
- [ ] The app shell shows active-trial and expired-trial status with a billing CTA.

**Out of scope:**
- Enforcing the entitlement result across every API route.
- Implementing usage metering or project/document/AI caps.
- Starting Polar checkout or changing webhook behavior.
- Adding transactional email reminders.

## Implementation notes

- Added a pure entitlement evaluator with read, write, upload, AI action, export, and billing-management allowances.
- Added a server-side current-user entitlement loader that combines Better Auth trial fields with Polar subscription records.
- Added an app-shell entitlement banner for dashboard and project workspaces.
- Added reusable blocked-action messaging for later enforcement issues.
