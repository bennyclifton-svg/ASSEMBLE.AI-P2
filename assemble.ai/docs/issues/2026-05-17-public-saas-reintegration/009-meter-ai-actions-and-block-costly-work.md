# 009 - Meter AI actions and block costly work

**Type:** AFK
**Triage category:** `enhancement`

## What to build

Add centralized AI action metering and apply it before model calls, AI workflow starts, and expensive background work. Trial users should receive a capped number of AI actions, paid users should receive plan-based allowances, and expired users should be blocked before cost is incurred.

## Agent Brief

> *This was generated by AI during triage.*

Add a central AI usage meter that checks entitlement state, project ownership when a project is involved, and the plan/trial AI action limit before model calls or action workflow execution. Record a reserved usage event before costly work, mark it succeeded after successful completion, and mark it failed when the costly call throws so failures are auditable without consuming the cap.

## Acceptance criteria

- [x] A usage meter records billable or capped AI actions against the authenticated user/workspace.
- [x] AI routes and workflow launch paths check ownership and AI allowance before model calls.
- [x] Trial users are blocked when the AI cap is reached.
- [x] Expired trial users are blocked before model or worker cost is incurred.
- [x] Paid users receive allowance according to the plan catalog.
- [x] Tests cover allowed AI action, over-cap block, expired-user block, paid-plan allowance, and failed action handling.
