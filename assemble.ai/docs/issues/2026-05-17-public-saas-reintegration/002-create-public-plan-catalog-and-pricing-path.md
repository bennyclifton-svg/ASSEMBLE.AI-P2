## Agent Brief

**Category:** enhancement
**Summary:** Create a single public SaaS plan catalog and make public pricing render from it.

**Current behavior:**
Plan and pricing data is split across several places. One Polar-facing plan definition includes Free, Starter, and Professional. A separate subscription tier definition also includes Free, Starter, and Professional. The public landing pricing section has its own hard-coded pricing cards, also including Free. The dedicated pricing page renders that landing pricing section and its FAQ still describes a Free tier as part of the plan comparison. Pricing links already include plan intent for paid plans in the URL, but the public register flow does not yet consume or validate that intent.

This drift is a launch risk for public SaaS because the product decision is now trial-first: public users choose Starter or Professional, start a no-card 14-day trial, and do not see a public Free plan after trial.

**Desired behavior:**
There should be one public SaaS plan catalog that is the source of truth for public pricing display, paid plan metadata, trial limits, paid limits, and Polar product mapping. Public pricing should render Starter and Professional from that catalog only. Free may remain as an internal fallback or legacy account state if needed, but it should not be offered as a public plan.

The catalog should make trial terms explicit: 14-day no-card trial, full feature access during trial, capped usage, and initial trial caps. Use one project, 100 documents, and 100 AI actions as the first explicit trial cap values unless a later product issue changes them. The shape should be testable and reusable by later signup, entitlement, billing, and usage-metering issues.

Plan CTAs for guests should preserve plan intent in the registration URL for Starter and Professional. Logged-in checkout behavior should continue to use the Better Auth/Polar slug model for paid plans. Public FAQ and landing/pricing copy should align with the catalog and no longer describe Free as a public plan.

**Key interfaces:**
- Public plan catalog - should expose paid public plans, trial metadata, paid limits, feature/display copy, and Polar product slug/product-id mapping in one place.
- Public pricing UI - should render plan cards from the catalog rather than local hard-coded plan arrays.
- Billing plan accessors - should either consume the shared catalog or be clearly adapted from it so paid plan metadata and Polar mapping do not drift.
- Subscription tier helpers - should no longer be a competing public pricing source; if kept, they should delegate to or align with the shared catalog.
- Register links - should carry a validated plan identifier for Starter or Professional.

**Acceptance criteria:**
- [x] A single plan catalog defines Starter and Professional display metadata, trial limits, paid limits, and Polar product identifiers or slugs.
- [x] The catalog records a no-card 14-day trial with full feature access and initial caps of one project, 100 documents, and 100 AI actions.
- [x] Public pricing renders from the shared plan catalog.
- [x] Public pricing no longer presents Free as a public plan.
- [x] Pricing CTAs include plan intent for Starter and Professional.
- [x] Public trial terms and FAQ copy match the catalog.
- [x] Existing logged-in paid checkout still uses Starter and Professional slugs compatible with Better Auth/Polar.
- [x] Tests cover the public plan list, removal of public Free, trial metadata, trial limits, paid limits, and Polar product mapping.

**Out of scope:**
- Starting trials on signup or storing trial fields on the auth user.
- Enforcing trial caps, usage metering, or expired-trial read-only behavior.
- Implementing Polar webhook handling or subscription source-of-truth cleanup.
- Adding transactional email, legal pages, account settings, or deployment docs.
- Removing every internal reference to a Free fallback state if it is still needed for existing local/private or billing behavior.
