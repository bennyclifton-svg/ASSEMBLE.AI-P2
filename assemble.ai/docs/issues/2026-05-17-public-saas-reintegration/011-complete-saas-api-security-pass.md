## What to build

Run the mandatory public SaaS security pass over project, upload, AI, billing, and admin APIs. Every route should either enforce authenticated ownership and entitlement through shared helpers, be explicitly public/read-only, or be documented as intentionally exempt.

## Agent Brief

> *This was generated by AI during triage.*

Use the shared access helpers from issue 010 as the enforcement path. Prioritise project routes that are currently unauthenticated or mutating, admin APIs that only check ordinary auth, and the existing upload/AI/billing gates. Produce a route audit document so covered routes, deliberate public routes, and follow-up candidates are visible.

## Acceptance criteria

- [x] All project APIs either require project ownership or are documented as intentionally public/non-project routes.
- [x] All mutating project APIs require writable entitlement.
- [x] Upload APIs require project ownership and upload allowance.
- [x] AI/action/workflow APIs require project ownership and AI allowance where they can incur cost or mutate data.
- [x] Billing APIs operate only on the authenticated user's customer/subscription records.
- [x] Admin APIs require super-admin authority, not ordinary authentication.
- [x] A route audit document or test inventory records covered routes and explicit exemptions.
- [x] Regression tests cover at least one cross-user access attempt in each major API family.
