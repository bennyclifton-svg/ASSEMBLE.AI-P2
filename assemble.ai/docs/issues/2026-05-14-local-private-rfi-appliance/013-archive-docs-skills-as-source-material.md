## Agent Brief

**Category:** enhancement
**Summary:** Reclassify the former markdown skill collection as archived/source material and remove misleading runtime-skill claims from repo documentation.

**Current behavior:**
The repo still contains a collection of 38 former skill markdown documents. Current strategy, README, and codebase-review docs already state that these files are source/reference material only, and a search of runtime code does not show an active loader for these markdown files. However, the collection's folder/name shape and several surrounding documents still make it easy for future contributors or agents to mistake the material for live runtime capabilities.

Documentation that currently implies live skills, desktop-harness storage, Chroma-based indexing, `project.db`, or file-watcher behaviour should be corrected, flagged as historical, or routed through an archive note.

The change should keep the runtime boundary explicit: do not add a skill loader, plugin system, file watcher, Chroma integration, SQLite/project.db path, or new runtime agent mechanism as part of this issue.
